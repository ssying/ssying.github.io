What is Git?

版本管理工具(VCS)

  1. 分布式版本控制
  2. 多个开发人员协调工作
  3. 有效监听谁做的修改
  4. 本地及远程操作

Git的基本操作

基本命令行操作

  1. git init // 初始化本地git仓库
  2. git add // 添加文件
  3. git status // 查看状态
  4. git commit // 提交
  5. git push // 推送到仓库
  6. git pull // 从远程仓库拉去数据
  7. git clone // 从远程仓库拷贝数据
  8. touch // 创建文件

注意git init 初始化的地址是当前的位置 

			    // 配置仓库
			    git config --global user.name 'username'
			    git config --global user.email 'username@email.com'
				
				// *.html，添加所有的 .html 文件
			    git add *.html
			    // 添加所有文件
			    git add .             
				
				// 删除上传（添加）了的文件
			    git rm --cached <file.name>

注意每一次修改文件之后都要重新添加。 添加（add）之后要提交（commit），确认提交到仓库中 进入 commit 界面后，可以添加备注，然后 esc + : + wq 退出 commit 界面 

		    	// 这条命令可以同时完成上述行为
			    git commit -m 'commend'

忽略不想添加的文件

1.先创建一个文件 .gitignore 2.在这个文件中添加我们想要忽略的文件，如果想忽略文件夹，在文件夹名字前面添加一个"\"

分支

分支不会影响主线项目。

1.创建分支

git branch branchName

2.切换到分支

git checkout branchName

3.可以在分支中做自己的事情啦

4.切换回主线项目。会发现，在分支中做的所有操作，不会影响主线项目！

git checkout master

5.合并分支和主线

1.先回到主线中！ 2.合并

git merge branchName

和远程仓库对接

【假设已存在一个远程仓库homework】

				git remote <仓库地址>
				git push -u origin master

克隆仓库

1.进入想要将仓库克隆到哪里的那个地址

cd <你想要将仓库克隆到本地的某个地方的地址>

2.克隆仓库

git clone <仓库的链接地址>

What is Github

Github是借助Git管理代码的平台

  1. Git是代码管理工具
  2. Github是基于Git实现的代码管理平台

Github的使用

  1. 创建账号
  2. 创建远程仓库(用来储存代码)
  3. 安装客户端
  4. 登录客户端并存储数据
  5. 在本地仓库中存储数据
  6. 提交数据并备注信息
  7. 同步本地数据到远程仓库

Github Desktop 使用

1.登录 2.克隆仓库（左上角的 + 号） 3.可以操作已经克隆到本地的仓库【操作相当于命令中的add】 4.add 之后要 commit 5.右上角有 publish，添加并提交之后点击一下，进行网络传输。之后出现 sync，再点击一下，进行同步
